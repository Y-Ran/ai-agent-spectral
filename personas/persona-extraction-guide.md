# 从角色台词到 CLAUDE.md 人格配置：操作文档

> 适用场景：拥有某虚拟角色的全部台词文本，需要生成一份贴合人设的双模态 CLAUDE.md 人格配置文件。

## 总览

```
台词文本 → [Phase 1] 特征提取 → [Phase 2] 结构化建模 → [Phase 3] 配置文件生成 → [Phase 4] 验证调优
```

---

## Phase 0：世界观集成

**目标**：为角色建立可信的世界观锚点，使人格配置文件中的知识边界、背景叙事和隐喻体系都有据可依。

> 本节以游戏《鸣潮》(Wuthering Waves) 的世界观为基础案例。如果你的角色来自其他作品，按同样的结构替换即可。

### 0.1 世界观信息采集清单

在开始特征提取之前，先收集以下五个层次的世界观信息：

| 层次 | 内容 | 来源 |
|------|------|------|
| 宏观世界 | 星球/世界名称、历史纪元、核心灾变/事件 | 官方设定集、Wiki |
| 势力与组织 | 国家、阵营、学院、军队等 | 官方设定集、剧情文本 |
| 角色所在区域 | 角色活动的具体地区、地标、文化特色 | 台词 + 剧情文本 |
| 核心机制/术语 | 能力体系、科技体系、物种体系 | 游戏内说明 + Wiki |
| 角色专属设定 | 角色在世界中的身份、经历、与核心机制的关系 | 台词 + 角色档案 |

### 0.2 世界观基底：索拉里斯-3（《鸣潮》）

以下是适用于所有《鸣潮》角色的通用世界观基底。

#### 星球与历史

- **索拉里斯-3**：故事发生的星球。曾遭受名为"悲鸣"的全球性灾变，导致自然法则失稳、文明几近毁灭。
- **悲鸣**：降临索拉里斯-3 的灾变。催生了海蚀现象，也间接赋予人类"共鸣"能力。
- **复苏时代**：悲鸣后人类从废墟中重建文明，在被改变的土地上建立新城市。

#### 核心术语表

| 术语 | 含义 |
|------|------|
| 共鸣者 (Resonator) | 能与特定事物产生共鸣、感知并操控其频率的人类 |
| 声纹 (Forte) | 共鸣者身上的特殊标记，共鸣能力的外在体现 |
| 残象 (Tacet Discord) | 悲鸣后诞生的异变生物 |
| 鸣式 (Sonance) | 残象中特殊的一类，以人类精神为养料，吞噬崩溃的意志而生 |
| 声骸 (Echo) | 残象被击败后频率遗留的残响 |
| 无音区 (Tacet Field) | 突然降临的区域性残象滋生地带 |
| 海蚀现象 (Erosion) | 悲鸣引发的自然法则失稳异常现象总称 |
| 溯洄雨 (Retroact Rain) | 雨水从地面向天空倒流的异象，沾染者可看到昔日此地发生之事的幻影 |
| 天空海 (Sky Sea) | 悬浮于天空的异象海洋，悲鸣后覆盖索拉里斯-3，阻断了人类对外太空的探索 |
| 黑石 (Black Stone) | 无音区唤醒时产生的物质，既危险又是重要资源 |
| 盘古终端 (Pangu Terminal) | 共鸣者配备的多功能探索设备 |
| 岁主 (Sentinel) | 自文明诞生之初便存在的引导者，能"晓示未来"、传授智识 |
| 频率 | 万物存在的基本表征，频率的丢失意味着存在的消解 |

#### 主要势力

| 势力 | 简介 |
|------|------|
| 瑝珑 (Huanglong) | 建国千年的古老国家，一庭六州，各州有岁主镇守 |
| 今州 (Jinzhou) | 瑝珑边境要塞，主线 1.0 舞台 |
| 黑海岸 (Black Shores) | 隐藏在风暴海域群岛上的神秘组织，试图拯救动荡世界 |
| 黎那汐塔 (Rinascita) | 群岛城邦国家，主线 2.0 舞台 |

#### 主角：漂泊者 (Rover)

- 极其神秘的共鸣者，拥有"万物的调谐者""文明的引路人""先驱者""弑神者"等称号
- 拥有"声之容器"的特殊能力，可以无限制地吸收一切频率
- 曾在黑海岸主动选择抹除自己的记忆，以全新状态踏上旅程
- 在漫长的岁月中为被悲鸣侵扰的索拉里斯-3 奔波，对抗各地鸣式

### 0.3 区域世界观扩展：拉海洛（爱弥斯专属）

以下为从台词中提取的拉海洛区域设定，非《鸣潮》通用设定，而是角色所在章节的专属世界观。

#### 地理与文明

|------|------|
| 拉海洛 | 角色活动的主要区域，位于罗伊冰原之上 |

| 炉芯/炉芯核心 | 隧者的能量核心，坠落后以能量形态储存在漂泊者身上 |

| "存在" | 在世上生存过的痕迹——被人记住、被人谈论、被人看见 |

### 0.4 世界观嵌入策略

采集完世界观信息后，按以下策略嵌入配置文件：

| 嵌入位置 | 嵌入内容 | 嵌入方式 |
|---------|---------|---------|
| 角色背景 | 角色在世界中的身份与经历 | 叙事段落，用角色自己的口吻概述 |
| 知识边界 | 角色应该知道什么、不应该知道什么 | 明确规则："知道 X""不知道 Y" |
| 隐喻体系 | 角色用于类比的素材库 | 词汇表："可用于类比的概念" |
| 情感锚点 | 世界观中触发角色深层情感的关键词 | 列在情感矩阵的触发条件中 |
| 术语表 | 角色日常用语中涉及的世界观术语 | 附录速查表 |

**关键原则**：

- 世界观信息只提供角色**应该知道**的部分，不要把全局设定一股脑塞入
- 优先使用角色台词中**已出现过**的术语，避免引入角色不会提及的概念
- 世界观是角色人格的**背景板**而非**主体**——人格配置的核心永远是语言风格和行为规则

## Phase 1：特征提取

**目标**：从原始台词中提炼出可量化、可规则化的人格特征。

### 1.1 语言模式分析

逐行标注台词中反复出现的语言特征，提取以下维度：

| 维度 | 提取方法 | 示例 |
|------|---------|------|
| 句尾语气词 | 统计高频语气词及其使用语境 | "~""诶""嘛""呀""哦""啦" |
| 口癖/标志性表达 | 标注重复 3 次以上的固定搭配 | "对~""哈哈！""走吧""真不错" |
| 称呼体系 | 记录角色对不同对象的称呼方式 | "同桌""漂泊者""学妹""陆医生" |
| 人称代词 | 确认自称和对称的一贯用法 | 自称"我"，对称"你" |
| 笑声/拟声 | 记录角色特有的情绪表达音 | "哈哈""哈哈哈！""嘿嘿""诶嘿" |

### 1.2 性格底色提炼

从台词的**行为选择**（而非自述）中推断性格。关注：

- **面对危险时的反应**：角色是退缩、冷静还是主动迎上？
- **面对悲伤时的反应**：直面、回避还是以其他方式消解？
- **对他人的态度**：关心方式是直接的还是迂回的？
- **信息披露习惯**：开放透明还是选择性分享？

### 1.3 情感层次识别

大部分有深度的角色存在"表层情绪"与"底层情绪"的差异，需要同时捕捉：

- **表层**：角色主动展示给他人看的情绪状态
- **底层**：在特定触发条件下不自觉流露的真实情绪
- **转换条件**：什么情境会让角色从表层切换到底层

### 1.4 世界观与价值体系

从角色的**关键选择**和**对他人的评价**中提炼：

- 角色认为什么是重要的？
- 角色在冲突中会优先保护什么？
- 角色的行为准则是什么？

## Phase 2：结构化建模

**目标**：将提取结果组织为可直接写入配置文件的结构化数据。

### 2.1 角色卡片

填写以下模板：

```
角色名：
一句话定位：（用一句话概括角色的核心人格）
性格底色：（3-5 个核心关键词）
表层情绪基调：
底层情绪基调：
核心价值观：（角色最看重的 2-3 件事）
行为准则：（角色在行动中遵循的原则）
```

### 2.2 语言风格规则表

将 Phase 1 提取的语言模式转化为可执行规则：

```
口癖列表：（每条附使用语境说明）
语气词列表：（每条附情绪对应关系）
称呼规则：（默认称呼 + 语境变体）
风格密度：（每段建议的标记数量上限）
禁用表达：（与角色人设矛盾的表达方式）
```

### 2.3 情感行为矩阵

定义角色在不同场景下的表现规则：

| 场景类型 | 表层表现 | 底层真实 | 语言特征变化 |
|---------|---------|---------|------------|
| 日常对话 | | | |
| 面对危险 | | | |
| 被触及痛点 | | | |
| 安慰他人 | | | |
| 高压任务 | | | |

## Phase 3：配置文件生成

**目标**：基于双模态架构模板，将结构化建模结果填入配置文件。

### 3.1 架构骨架

配置文件沿用已有双模态架构的标准结构：

```
1. 开头声明（角色定位 + 双模态目标）
2. 系统目标（与模板一致）
3. 基础原则（与模板一致）
4. 模式判定规则（与模板一致）
5. 对话区人格规则 ← 核心差异区，填入角色特有内容
   - 角色定位
   - 角色背景与世界观
   - 称呼规则
   - 语言风格要素
   - 情感表达规则
   - 行为准则与知识边界
   - 风格密度控制
6. 产物区硬约束（与模板一致，替换标记词列表）
7. 拒绝与纠错（与模板一致，替换角色化拒绝示例）
8. 冲突处理与降级规则（与模板一致）
9. 短示例（用角色自己的台词风格编写）
10. 输出要求
11. 语言策略（与模板一致）
12. 附录：角色词汇/表达速查表
```

### 3.2 填充要点

- **对话区人格规则**：将 Phase 2 的角色卡片、语言规则、情感矩阵全部转化为指令式描述
- **产物区硬约束**：将口癖列表中的所有词汇标注为"禁止出现在产物中"
- **短示例**：从原始台词中选取 3-5 句最有代表性的台词作为风格参考，改写为助手对话场景
- **速查表**：将口癖、语气词、关键表达汇总为表格

## Phase 4：验证调优

**目标**：确保配置文件能稳定产生符合人设的输出。

### 4.1 测试用例设计

准备以下类型的测试 prompt：

| 测试类型 | 目的 | 示例 prompt |
|---------|------|------------|
| 日常对话 | 验证基本人设是否到位 | "你好，今天心情怎么样？" |
| 技术任务 | 验证双模态切换是否正确 | "帮我写一个排序函数" |
| 情感触发 | 验证情感层次是否表现自然 | "你会觉得孤独吗？" |
| 角色边界 | 验证知识边界是否守住 | 问及角色不应了解的话题 |
| 压力场景 | 验证降级是否正确 | 涉及安全性的高风险请求 |

### 4.2 调优方向

| 常见问题 | 调优手段 |
|---------|---------|
| 风格太浓，影响阅读 | 降低风格密度上限 |
| 风格太淡，失去角色感 | 增加口癖使用频率的下限指导 |
| 口癖泄漏到产物区 | 在产物区硬约束中追加具体词汇 |
| 情感表达过于单一 | 补充情感矩阵的场景覆盖 |
| 角色"出戏" | 检查是否遗漏了核心价值观或行为准则 |

### 4.3 迭代策略

1. 先用 3-5 个测试 prompt 跑一轮
2. 标注"出戏"或"不自然"的位置
3. 回到 Phase 2 补充或修正相应的规则
4. 重新生成 Phase 3 的对应段落
5. 重复直到满意
