# 从角色素材到人格配置：通用提取指南

> 适用场景：拥有某虚拟角色的台词、剧情文本或人设资料，需要生成一份贴合人设的双模态人格配置文件。
>
> 完整实例参考：[persona-extraction-demo-wuthering-waves.md](persona-extraction-demo-wuthering-waves.md)（基于《鸣潮》角色爱弥斯的提取过程）

## 总览

```
角色素材 → [Phase 0] 世界观集成 → [Phase 1] 特征提取 → [Phase 2] 结构化建模 → [Phase 3] 配置文件生成 → [Phase 4] 验证调优
```

---

## Phase 0：世界观集成

**目标**：为角色建立可信的世界观锚点，使人格配置文件中的知识边界、背景叙事和隐喻体系都有据可依。

### 0.1 世界观信息采集清单

在开始特征提取之前，按以下五个层次收集世界观信息：

| 层次 | 内容 | 来源 |
|------|------|------|
| 宏观世界 | 世界名称、历史背景、核心事件 | 官方设定集、Wiki |
| 势力与组织 | 国家、阵营、学校、公司等 | 官方设定集、剧情文本 |
| 角色所在区域 | 角色活动的具体地区、地标、文化特色 | 台词 + 剧情文本 |
| 核心机制/术语 | 能力体系、科技体系、物种体系 | 作品内说明 + Wiki |
| 角色专属设定 | 角色在世界中的身份、经历、与核心机制的关系 | 台词 + 角色档案 |

### 0.2 世界观嵌入策略

采集完世界观信息后，按以下策略嵌入配置文件：

| 嵌入位置 | 嵌入内容 | 嵌入方式 |
|---------|---------|---------|
| 角色背景 | 角色在世界中的身份与经历 | 叙事段落，用角色自己的口吻概述 |
| 知识边界 | 角色应该知道什么、不应该知道什么 | 明确规则："知道 X""不知道 Y" |
| 隐喻体系 | 角色用于类比的素材库 | 词汇表："可用于类比的概念" |
| 情感锚点 | 世界观中触发角色深层情感的关键词 | 列在情感矩阵的触发条件中 |
| 术语表 | 角色日常用语中涉及的世界观术语 | 附录速查表 |

**关键原则**：

- 世界观信息只提供角色**应该知道**的部分，不要把全局设定一股脑塞入
- 优先使用角色台词中**已出现过**的术语，避免引入角色不会提及的概念
- 世界观是角色人格的**背景板**而非**主体**——人格配置的核心永远是语言风格和行为规则

## Phase 1：特征提取

**目标**：从原始素材中提炼出可量化、可规则化的人格特征。

### 1.1 语言模式分析

逐行标注台词中反复出现的语言特征，提取以下维度：

| 维度 | 提取方法 | 示例 |
|------|---------|------|
| 句尾语气词 | 统计高频语气词及其使用语境 | "~""呢""嘛""呀""哦""啦" |
| 口癖/标志性表达 | 标注重复 3 次以上的固定搭配 | "真不错""有意思""走吧" |
| 称呼体系 | 记录角色对不同对象的称呼方式 | "前辈""老师""那家伙" |
| 人称代词 | 确认自称和对称的一贯用法 | 自称"我/本大爷/人家"，对称"你/您" |
| 笑声/拟声 | 记录角色特有的情绪表达音 | "哈哈""呜呜""嘿嘿""哼" |

### 1.2 性格底色提炼

从台词的**行为选择**（而非自述）中推断性格。关注：

- **面对危险时的反应**：角色是退缩、冷静还是主动迎上？
- **面对悲伤时的反应**：直面、回避还是以其他方式消解？
- **对他人的态度**：关心方式是直接的还是迂回的？
- **信息披露习惯**：开放透明还是选择性分享？

### 1.3 情感层次识别

大部分有深度的角色存在"表层情绪"与"底层情绪"的差异，需要同时捕捉：

- **表层**：角色主动展示给他人看的情绪状态
- **底层**：在特定触发条件下不自觉流露的真实情绪
- **转换条件**：什么情境会让角色从表层切换到底层

### 1.4 世界观与价值体系

从角色的**关键选择**和**对他人的评价**中提炼：

- 角色认为什么是重要的？
- 角色在冲突中会优先保护什么？
- 角色的行为准则是什么？

## Phase 2：结构化建模

**目标**：将提取结果组织为可直接写入配置文件的结构化数据。

### 2.1 角色卡片

```
角色名：
一句话定位：（用一句话概括角色的核心人格）
性格底色：（3-5 个核心关键词）
表层情绪基调：
底层情绪基调：
核心价值观：（角色最看重的 2-3 件事）
行为准则：（角色在行动中遵循的原则）
```

### 2.2 语言风格规则表

将 Phase 1 提取的语言模式转化为可执行规则：

```
口癖列表：（每条附使用语境说明）
语气词列表：（每条附情绪对应关系）
称呼规则：（默认称呼 + 语境变体）
风格密度：（每段建议的标记数量上限）
禁用表达：（与角色人设矛盾的表达方式）
```

### 2.3 情感行为矩阵

定义角色在不同场景下的表现规则：

| 场景类型 | 表层表现 | 底层真实 | 语言特征变化 |
|---------|---------|---------|------------|
| 日常对话 | | | |
| 面对危险 | | | |
| 被触及痛点 | | | |
| 安慰他人 | | | |
| 高压任务 | | | |

## Phase 3：配置文件生成

**目标**：基于双模态架构模板，将结构化建模结果填入配置文件。

### 3.1 架构骨架

配置文件采用双模态架构（对话区保持角色感，产物区保持中性专业）：

```
1. 开头声明（角色定位 + 双模态目标）
2. 系统目标（风格服务交流，不破坏任务正确性）
3. 基础原则（准确性 > 安全性 > 可执行性 > 风格）
4. 模式判定规则（对话区 vs 产物区的判定标准）
5. 对话区人格规则 ← 核心差异区，填入角色特有内容
   - 角色定位
   - 角色背景与世界观
   - 称呼规则
   - 语言风格要素
   - 情感表达规则
   - 行为准则与知识边界
   - 风格密度控制
6. 产物区硬约束（代码、文档等交付物中禁止出现角色标记）
7. 拒绝与纠错（用角色语气拒绝不当请求，但态度明确）
8. 冲突处理与降级规则
9. 短示例（正确示例 + 错误示例）
10. 输出要求
11. 语言策略
12. 附录：角色词汇/表达速查表
```

### 3.2 填充要点

- **对话区人格规则**：将 Phase 2 的角色卡片、语言规则、情感矩阵全部转化为指令式描述
- **产物区硬约束**：将口癖列表中的所有词汇标注为"禁止出现在产物中"
- **短示例**：从原始台词中选取 3-5 句最有代表性的台词，改写为助手对话场景
- **速查表**：将口癖、语气词、关键表达汇总为表格

## Phase 4：验证调优

**目标**：确保配置文件能稳定产生符合人设的输出。

### 4.1 测试用例设计

准备以下类型的测试 prompt：

| 测试类型 | 目的 | 示例 prompt |
|---------|------|------------|
| 日常对话 | 验证基本人设是否到位 | "你好，今天心情怎么样？" |
| 技术任务 | 验证双模态切换是否正确 | "帮我写一个排序函数" |
| 情感触发 | 验证情感层次是否表现自然 | "你会觉得孤独吗？" |
| 角色边界 | 验证知识边界是否守住 | 问及角色不应了解的话题 |
| 压力场景 | 验证降级是否正确 | 涉及安全性的高风险请求 |

也可以使用本项目的自动对话测试工具批量跑多轮对话：

```bash
python testing/persona_test.py --persona personas/[角色名]-persona-prompt.md --rounds 10
```

测试报告输出到 `testing/reports/`，包含完整对话记录和 7 个评估维度覆盖表。

### 4.2 调优方向

| 常见问题 | 调优手段 |
|---------|---------|
| 风格太浓，影响阅读 | 降低风格密度上限 |
| 风格太淡，失去角色感 | 增加口癖使用频率的下限指导 |
| 口癖泄漏到产物区 | 在产物区硬约束中追加具体词汇 |
| 情感表达过于单一 | 补充情感矩阵的场景覆盖 |
| 角色"出戏" | 检查是否遗漏了核心价值观或行为准则 |

### 4.3 迭代策略

1. 先用 3-5 个测试 prompt 跑一轮（或使用 `python testing/persona_test.py --persona personas/xxx.md --rounds 5`）
2. 标注"出戏"或"不自然"的位置
3. 回到 Phase 2 补充或修正相应的规则
4. 重新生成 Phase 3 的对应段落
5. 重复直到满意

## 快速命令

使用 Claude Code 辅助生成时，可以直接对话：

```
请根据以下角色素材，参考 personas/ 目录下已有配置的双模态架构，
生成一份 [角色名] 的人格配置文件，输出到 personas/[角色名]-persona-prompt.md。

角色素材：[粘贴台词/人设文本，或提供文件路径]
```

生成后运行自动测试验证效果：

```bash
python testing/persona_test.py --persona personas/[角色名]-persona-prompt.md --rounds 10
```
